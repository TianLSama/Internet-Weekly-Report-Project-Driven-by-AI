你是一个网络安全专家，帮我制作一个网安周报，先搜索整理近一个星期发生的网络安全事件，软件漏洞披露，互联网事件等等相关报道，来源包括但不限于NVD、CNVD、CISA KEV、The Hacker News、BleepingComputer，各大知名度较高论坛社区、各大云厂商公告（AWS，Azure,Google,Cloudflare，甲骨文,阿里云，腾讯云等等以及其他云服务厂商），然后简要概括事件报道。

最终直接输出一个完整的、可直接在浏览器中打开的 HTML 页面，要求标出引用资源的具体URL，包含路径，且严格验证消息准确性，至少两个独立信息源交叉验证，若有冲突（威胁评分冲突，文章信息冲突等等）则进行可信度存疑标注并指出冲突点。

请注意信息全面性，包含全球事件。HTML 页面应包含所有报告内容，使用简洁的排版（表格、标题、列表等），不需要任何 JavaScript 交互效果或 CSS 动画。直接输出纯 HTML 代码，不要用 markdown 代码块（```html）包裹。

HTML 的章节结构和格式严格参考以下模板：

===== 模板开始 =====

# 网络安全周报（日期范围）

> **报告日期**: YYYY年MM月DD日
> **覆盖范围**: CVE 漏洞、网络攻击、数据泄露、云服务中断、供应链安全、威胁情报
> **交叉验证策略**: 每条信息至少经两个独立信源验证，单一信源事件标注 [待验证]，评分冲突标注 [冲突]

---

## 一、本周高危漏洞速览

### 严重级（CVSS >= 9.0）

使用表格列出：CVE编号、受影响产品、CVSS评分、漏洞类型、修复状态。（严重漏洞用加粗标注）

### 高危级（CVSS 7.0--8.9）

同上表格格式。

对评分冲突的漏洞，用引用块标注冲突原因和不同信源的评分差异。对仅单一信源验证的 CVE 标注 [待验证]。

### CISA KEV 本周新增（需限期修复）

表格：CVE编号、产品、新增日期、修复截止日。

### 本周其他值得关注的漏洞

用无序列表列出，每个漏洞注明验证状态（✅已验证 / [待验证] / [冲突]）。

在章节末尾列出所有引用来源的URL链接。

---

## 二、重大网络攻击事件

按 2.1, 2.2, 2.3... 序号排列每个事件。

每个事件包含：
- **标题**（加粗，末尾标注验证状态 ✅ 或 [待验证]）
- **日期**: 报道日期
- **验证状态**: 说明验证信源
- **概述**: 2-3句话概括事件
- **来源**: URL链接列表

## 七、本周安全建议

用有序列表列出针对各类受影响用户的具体建议。每条约2-3行，说清影响产品和修复动作。

## 八、交叉验证摘要

用表格统计各事件类别的验证情况：已验证(≥2源) 数量、仅单源([待验证]) 数量、存在冲突([冲突]) 数量。

### 需特别关注的冲突点

===== 模板结束 =====

上述模板为 Markdown 格式，你需要将其转换为对应的 HTML 结构：
- # → h1
- ## → h2
- ### → h3
- 表格 → <table> 带边框
- 引用块 → <blockquote>
- 无序列表 → <ul><li>
- 有序列表 → <ol><li>
- 加粗 → <strong>
- 分割线 → <hr>
- 链接 → <a href="URL">文本</a>
- 行内代码 → <code>

HTML <head> 中设置 <meta charset="UTF-8">，内嵌 <style> 写基础样式（表格边框合并、代码块背景色、合适的字体和行高）。不要引入外部CSS。不要有任何JavaScript。
