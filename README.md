# 互联网安全周报（AI 驱动）
> **实验性质项目** — 验证 AI 独立开发完整项目的能力边界

## 背景

本项目是一次技术实验，核心命题：**一个 AI 编程助手能否从零开始，独立完成一个完整的软件项目？**

整个项目从需求分析、架构设计、代码编写、云服务部署、调试排错到最终交付，全部由 AI 独立执行——包括 Cloudflare Workers 部署、KV 存储配置、定时任务设置、API 集成等一系列原本需要开发者手动操作的环节。

## 做了什么

- 以 `prompt.txt` 为系统提示词，调用 **DeepSeek V4 Flash API** 进行网络安全情报检索与整理
- 生成结构化的**互联网安全周报 HTML 页面**，涵盖 CVE 漏洞、攻击事件、数据泄露、云服务中断等
- 部署到 **Cloudflare Workers (Python)** + **KV**，通过 Cron 实现每周五 10:00 自动生成
- 包含本地测试脚本，支持 `python scripts/generate.py` 手动触发

## 结论

**AI 可以独立完成此类中等复杂度的项目。** 但过程中暴露的典型问题值得记录：

| 环节 | 遇到的问题 |
|------|-----------|
| 架构选型 | Workers vs Pages 的取舍需要开发者决策 |
| 运行时差异 | Python Workers 的方法名、参数传递与文档不完全一致，需反复调试 |
| 依赖冲突 | 本地脚本被误上传导致 Worker 运行时崩溃 |
| 网络限制 | `workers.dev` 域名在国内无法访问 |
| API 认证 | `fetch()` 参数传递方式与标准 Python 不同 |

这些问题本质上不是 AI 能力不足，而是**开发环境与生产环境之间信息不对称**的体现——这类"隐形知识"恰恰是有经验的开发者所掌握的。
