
# QWEN.md — 网络安全周报（AI 驱动）项目指南

## 项目概述

本项目是一个 **AI 驱动的网络安全周报生成系统**。通过向 AI 提供结构化的提示词（`prompt.txt`），自动搜集、整理过去一周全球网络安全事件、漏洞披露、云服务中断等信息，最终生成一份格式规范、来源可追溯、经过交叉验证的中文 `.md` 周报。

- **语言**: 中文（报告输出），英文（部分技术术语保留）
- **覆盖范围**: CVE 漏洞、网络攻击、数据泄露、云服务中断、供应链安全、威胁情报
- **核心原则**: 每条信息至少经两个独立信源交叉验证；单一信源标注 `[待验证]`；评分冲突标注 `[冲突]`

## 关键文件说明

### `prompt.txt`
核心提示词文件，定义了 AI 的角色、任务、信息来源、输出格式和验证规则。每次生成周报时以此作为系统提示。关键约束：
- 信息来源包括 NVD、CNVD、CISA KEV、The Hacker News、BleepingComputer、各大云厂商公告等
- 必须至少有**两个独立信源交叉验证**
- 输出为 Markdown，标明引用 URL 完整路径
- 避免过多 emoji

### `example.md`
一份完整的周报范例，展示了最终输出的格式规范。包含以下章节：
1. **高危漏洞速览** — CVE 表格（含 CVSS、类型、状态、验证标记）
2. **重大网络攻击事件** — 按事件编号，含日期、验证状态、概述、来源 URL
3. **数据泄露与隐私事件** — 同上结构
4. **供应链攻击** — 同上结构
5. **云服务与互联网中断** — Azure / Cloudflare / AWS / GCP 中断记录
6. **威胁情报与行业动态** — 行业报告、研究动态
7. **本周安全建议** — 按受影响产品分组的行动建议
8. **交叉验证摘要** — 统计表格 + 冲突说明 + 限制说明

## 使用方式

1. 将 `prompt.txt` 作为系统提示词提供给 AI
2. AI 按照提示词搜索、整理、交叉验证当周安全事件
3. 输出 `.md` 文件到项目目录，命名格式可参考：`网络安全周报_YYYY-MM-DD.md`
4. 比对 `example.md` 确保格式一致

## 验证标记说明

| 标记 | 含义 |
|------|------|
| `✅` | 至少两个独立信源验证通过 |
| `[待验证]` | 仅单一信源报道，建议人工二次确认 |
| `[冲突]` | 不同信源之间存在信息冲突（如 CVSS 评分差异），已标注冲突点 |

## 注意事项

- 本项目是纯文档/提示词项目，无代码。所有操作围绕提示词优化和报告模板迭代展开。
- 周报中的 URL 链接可能随时间失效（如示例中部分 Ars Technica 链接返回 404），这是信息来源可追溯性的固有限制。
- `example.md` 中标记了信源可访问性问题（BleepingComputer 403、The Register 超时等），生成新报告时需注意网络可达性。
